如何把本地文件与gitlab里面的项目联系起来，并可以上传 1、下载git 2、ssh-keygen -t rsa -C "gitlab的邮箱" ：就是在本地~/.ssh文件夹里面写入文件id_rsa（密钥）,ir_rsa.pub（公钥）。由于本地git仓库和github仓库之间传输是要通过ssh加密的，所以需要设置。 3、cat ~/.ssh/id_rsa.pub：查看公钥，并复制公钥复制到个人gitlab里面，settings->SSH and GPG keys->New SSH key里面。 4、ssh -T git@github.com：查看是否设置成功。 5、在本地文件夹中 git init生成.git文件。 6、git remote add origin gitlab里面的项目 7、git pull --rebase origin master：从6步中设置的gitlab项目里面先把内容拉下来。 8、git add . git commit -m "" git push -u origin master:就可以把本地的代码上传到gitlab里面。以后就只需要git push origin master就行。 廖雪峰：https://www.liaoxuefeng.com/wiki/0013739516305929606dd18361248578c67b8067c8c017b000 git命令： 1、git status： 2、git diff file:显示file文件改变之处 3、git log [--pretty=oneline] ：查看提交历史，确定回退版本，中括号里面显示简化版提交历史。 4、git reset --hard HEAD^：HEAD代表当前文件，HEAD^代表上一个版本，HEAD~100代表100次之前的版本。 5、git reset --hard 3642fvc5：回到3642fvc5这个commit id的版本 6、git reflog：查看历史命令 7、git diff HEAD -- file ：查看工作区和版本库文件差别 8、git checkout -- file：丢弃工作区对file文件的修改 9、git reset HEAD File：把暂存区修改取消，后面可以接命令8，再把工作区修改取消。 10、git stash：把当前工作现场储存起来，等后面修改好bug再回来继续工作。 11、git stash pop：回到git stash保存之处。 12、git branch -D test：强行删除test分支，并丢失分支修改内容。 13、git remote [-v]：查看远程库的信息，-v显示更详细的信息 14、git checkout -b dev origin/dev：其他人在你的dev分支上开发，就需要创建远程origin的dev分支到本地。 15、git branch --set-upstream dev origin/dev：把本地的dev分支与远程origin/dev分支链接起来。 16、git pull：把远程分支里面的内容拉取到本地。 17、git tag v1.0：默认标签是在最新提交的commit上的， 18、git log --pretty=oneline --abbrev-commit：找到历史提交的commit id 19、git tag v2.0 5654151 ：对commit id 5654151打上v2.0标签。 20、git tag：查看标签 21、git show v1.0：查看标签信息 22、git tag -a v2.0 -m "test" 325641：创建带说明的标签，-a标签名，-m指定说明文字。 23、git branch |grep 'test'|xargs git branch -d：删除包含test字符串的branch分支。
